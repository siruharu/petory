# 반려동물 건강/생활 관리 앱 MVP 범위 정의

작성일: 2026-03-23  
기준 문서: `docs/research.md`  
목적: 2~4주 안에 구현 가능한 앱 우선 MVP 범위를 정의한다.  
주의: 이 문서는 범위 정의 문서다. 구현 세부 설계와 기술 선택은 이후 사람이 결정한다.

## 1. 범위 정의 원칙

이번 MVP는 다음 원칙으로 제한한다.

- 앱 우선으로 정의한다.
- “기억에 의존하던 반려동물 관리”를 “앱 안의 일정과 기록”으로 옮기는 데 집중한다.
- 로그인, 반려동물 등록, 일정, 알림, 기록 기능을 제품의 중심으로 둔다.
- 2~4주 안에 구현 가능한 범위만 포함한다.
- 여러 확장 가능성보다 사용자가 첫날부터 바로 쓰는 핵심 흐름을 우선한다.

## 2. MVP 목표

이 MVP가 해결해야 하는 핵심 문제는 아래 두 가지다.

- 보호자가 반려동물 관리 일정을 잊어버리는 문제
- 접종, 투약, 병원 방문 같은 핵심 기록이 흩어져 있는 문제

따라서 MVP의 핵심 가치는 다음으로 정의한다.

- 계정을 만들고 로그인할 수 있다.
- 반려동물을 등록할 수 있다.
- 반려동물별로 중요한 일정과 리마인더를 만들 수 있다.
- 실제로 실행한 건강 기록을 남길 수 있다.
- 오늘 해야 할 일과 최근 기록을 앱 안에서 바로 확인할 수 있다.

## 3. MVP 사용자 가정

이번 MVP는 가장 넓고 현실적인 초기 사용자로 아래 집단을 기준으로 잡는다.

- 개 또는 고양이 1마리를 키우는 일반 보호자
- 접종, 구충/예방약, 약 복용, 병원 방문 정도의 기본 건강 관리를 원하는 사용자
- 복잡한 협업보다는 개인 사용 중심인 사용자

이번 범위에서 일부러 제외하는 사용자 상황은 다음과 같다.

- 가족 공동 양육을 전제로 한 멀티유저 협업
- 펫시터/호텔/병원과의 외부 공유가 핵심인 사용 패턴
- 웨어러블/GPS 연동을 기대하는 고관여 사용자
- 만성질환의 정밀 추적을 우선 가치로 두는 사용자

## 4. 앱 우선 MVP 핵심 사용자 흐름

이 MVP에서 앱의 가장 중요한 흐름은 아래 순서다.

1. 회원가입 또는 로그인
2. 첫 반려동물 등록
3. 기본 일정 생성
4. 홈에서 오늘 일정 확인
5. 일정 완료 처리 또는 기록 추가
6. 최근 기록 확인

이 흐름이 자연스럽게 이어져야 MVP로서 의미가 있다.

## 5. 필수 기능

### 5.1 로그인

MVP에서 반드시 필요한 이유:

- 반려동물 데이터와 일정/기록을 사용자 단위로 저장해야 함
- 향후 다기기 사용을 위한 최소 기반이 됨

포함 범위:

- 이메일 기반 회원가입
- 이메일 기반 로그인
- 로그아웃
- 최소한의 인증 상태 유지

이번 MVP에서 의도적으로 단순화할 부분:

- 소셜 로그인 제외
- 다중 계정 연결 제외
- 복잡한 권한 체계 제외

### 5.2 반려동물 등록

MVP에서 반드시 필요한 이유:

- 일정, 알림, 기록의 기준 엔티티가 반려동물이기 때문

포함 범위:

- 반려동물 추가
- 반려동물 기본 정보 수정
- 반려동물 목록 확인
- 반려동물 전환

MVP 필수 입력 항목:

- 이름
- 종
- 품종 또는 자유 입력 품종명
- 성별
- 중성화 여부
- 생년월일 또는 나이
- 몸무게
- 메모

선택 입력으로 둘 수 있는 항목:

- 프로필 사진
- 알러지/주의사항

이번 MVP에서 제외:

- 마이크로칩 번호
- 등록번호
- 병원/응급병원 상세 관리
- 여러 보호자 연결

### 5.3 일정

MVP에서 반드시 필요한 이유:

- 사용자가 앱을 지속적으로 쓰게 만드는 핵심 가치가 일정 관리이기 때문

포함 범위:

- 반려동물별 일정 생성
- 일정 목록 조회
- 일정 상세 조회
- 일정 수정
- 일정 완료 처리
- overdue 상태 표시

MVP에서 지원할 일정 유형:

- 예방접종
- 예방약/구충/심장사상충 등 주기성 건강 일정
- 약 복용
- 병원 방문
- 자유 일정

MVP에서 지원할 일정 속성:

- 제목
- 일정 유형
- 대상 반려동물
- 예정일
- 반복 여부
- 메모
- 완료 여부

반복 규칙은 MVP에서 다음 수준으로 제한하는 것이 적절하다.

- 반복 없음
- 매일
- 매주
- 매월
- 사용자 지정 간격은 제외하거나 매우 단순한 형태로 제한

이번 MVP에서 제외:

- 복잡한 캘린더 뷰
- 담당자 할당
- 세부 체크리스트
- 다단계 승인/완료 프로세스

### 5.4 알림

MVP에서 반드시 필요한 이유:

- 일정 기능은 알림이 없으면 실사용 가치가 크게 떨어짐

포함 범위:

- 앱 푸시 알림 또는 앱 내부 알림 중 하나를 중심으로 제공
- 일정 due 시점 알림
- overdue 상태 시각화

알림 정책은 MVP에서 단순하게 가져간다.

- 일정당 기본 알림 1회
- 사용자가 알림 시간을 선택할 수 있으면 좋지만, 초기에는 고정 규칙도 가능
- overdue는 별도 푸시보다 앱 홈 강조가 우선

이번 MVP에서 제외:

- SMS
- 이메일 알림
- 다중 사전 알림
- 세밀한 알림 규칙 설정
- 사용자별 알림 채널 커스터마이징

### 5.5 기록

MVP에서 반드시 필요한 이유:

- “할 일 관리 앱”이 아니라 “반려동물 관리 앱”으로 느껴지게 하는 핵심이 기록이기 때문

포함 범위:

- 일정 완료 시 기록 남기기
- 기록 직접 추가
- 반려동물별 기록 목록
- 최근 기록 확인

MVP에서 지원할 기록 유형:

- 접종 기록
- 투약 기록
- 병원 방문 기록
- 체중 기록
- 메모 기록

기록 필수 속성:

- 반려동물
- 기록 유형
- 날짜
- 메모

기록 유형별 권장 최소 필드:

- 접종 기록: 접종명, 날짜, 메모
- 투약 기록: 약 이름, 날짜, 메모
- 병원 방문 기록: 방문일, 메모
- 체중 기록: 측정일, 체중
- 메모 기록: 작성일, 메모

이번 MVP에서 제외:

- 사진/PDF 첨부
- 검사 결과 구조화 입력
- 증상 severity 추적
- 복약 계획과 실제 투약 로그의 정교한 분리
- 리포트 export

### 5.6 홈 화면

MVP에서 반드시 필요한 이유:

- 사용자는 앱 첫 화면에서 “오늘 뭘 해야 하는지”를 즉시 이해해야 함

포함 범위:

- 오늘 예정된 일정
- overdue 일정
- 최근 기록
- 현재 선택된 반려동물 표시

이번 MVP에서 제외:

- 주간/월간 통계
- 건강 인사이트
- 추천 콘텐츠

## 6. 제외 기능

2~4주 MVP에서 명확히 제외해야 하는 기능은 아래와 같다.

### 6.1 협업/공유

- 가족 초대
- 공동 양육
- 시터 공유 링크
- 병원 공유 링크

제외 이유:

- 권한과 데이터 공유 모델이 추가되면 복잡도가 급격히 상승함

### 6.2 고급 건강 관리

- 만성질환 전용 관리 플로우
- 증상 구조화 추적
- 음수량/식욕/배변의 정밀 로그
- 수의사용 요약 리포트 자동 생성

제외 이유:

- 초기 MVP에서는 범위가 너무 넓고 입력 부담이 커짐

### 6.3 고급 알림

- 반복 예고 알림 여러 단계
- 사용자별 알림 정책
- 일정 유형별 세부 규칙

제외 이유:

- 기본 due 알림만으로도 MVP 가치 검증이 가능함

### 6.4 문서/미디어

- 사진 업로드
- PDF/문서 첨부
- OCR

제외 이유:

- 저장소, 업로드, 미디어 처리, 데이터 구조 복잡도가 늘어남

### 6.5 외부 연동

- 웨어러블
- GPS
- 병원 예약 연동
- 보험/결제
- 커머스

제외 이유:

- 외부 시스템 의존성과 제품 범위가 커짐

## 7. 2~4주 기준 현실적인 MVP 범위 제안

### 7.1 2주에 가까운 최소 범위

정말 타이트하게 자르면 아래 구성이 최소선이다.

- 이메일 로그인
- 반려동물 1마리 이상 등록
- 기본 일정 생성/수정/완료
- due/overdue 표시
- 기본 알림 1회
- 접종/투약/병원 방문/메모 기록
- 홈에서 오늘 일정과 최근 기록 확인

이 범위는 “잊지 않게 하고, 남겨두게 한다”는 핵심 가치를 검증할 수 있다.

### 7.2 4주까지 허용 가능한 확장 범위

일정이 4주에 가까우면 아래를 추가 검토할 수 있다.

- 반려동물 여러 마리 지원 강화
- 체중 기록 전용 입력 UI
- 일정 반복 규칙 확대
- 일정 완료 시 자동 기록 생성 UX 개선
- 알림 시간 선택
- 반려동물 프로필 사진

이 항목들은 가치가 높지만, MVP 핵심이 없어도 되는 기능보다 뒤에 둬야 한다.

## 8. 우선순위

### P0: 없으면 MVP가 성립하지 않음

- 로그인
- 반려동물 등록
- 일정 생성/조회/수정/완료
- due/overdue 상태 표시
- 기본 알림
- 기록 추가/조회
- 홈 화면의 오늘 일정/최근 기록

### P1: 4주 범위에서 검토 가능

- 반려동물 사진
- 여러 마리 사용성 개선
- 체중 입력 전용 UX
- 알림 시간 선택
- 일정 반복 규칙 약간 확장

### P2: 후속 릴리즈로 넘겨야 함

- 협업/공유
- 첨부파일
- OCR
- 리포트
- 웨어러블/GPS
- 고급 건강 분석

## 9. 화면 단위 기준 범위

앱 우선 MVP는 아래 화면 단위로 정의하는 것이 적절하다.

- 로그인 화면
- 회원가입 화면
- 반려동물 등록/수정 화면
- 홈 화면
- 일정 목록/상세/생성 화면
- 기록 목록/생성 화면
- 설정 또는 로그아웃 진입 화면

이번 MVP에서 굳이 분리하지 않아도 되는 화면:

- 고급 캘린더 화면
- 통계 화면
- 공유 관리 화면
- 알림 설정 세부 화면

## 10. 최종 범위 선언

이번 MVP는 “개인 사용자가 앱에서 로그인하고, 반려동물을 등록한 뒤, 중요한 건강/관리 일정을 만들고, 알림을 받고, 실제 실행한 기록을 남길 수 있는 최소 제품”으로 정의한다.

포함 기능은 다음 다섯 축에 집중한다.

- 로그인
- 반려동물 등록
- 일정
- 알림
- 기록

제외 기능은 다음 원칙으로 정리한다.

- 협업은 제외
- 첨부와 문서 처리는 제외
- 외부 연동은 제외
- 고급 건강 분석은 제외
- 복잡한 알림 설정은 제외

이 범위가 2~4주 MVP로 가장 현실적이며, 이후 확장은 `공유`, `첨부`, `정교한 건강 기록`, `멀티유저`, `웨어러블` 순으로 검토하는 것이 자연스럽다.
