# 반려동물 건강/생활 관리 앱 연구

작성일: 2026-03-23  
목적: 반려동물 건강/생활 관리 앱의 사용자, 문제 시나리오, 기능 범위, 데이터 구조, 기록/알림 패턴, MVP 범위와 확장 가능성을 매우 상세히 정리한다.  
주의: 이 문서는 연구 문서다. 설계 방향, 우선순위, 기술 선택은 이후 사람이 결정해야 한다.

## 1. 연구 범위와 관찰 기준

이 문서는 다음 관점으로 반려동물 관리 앱을 분석한다.

- 일상 관리: 급여, 산책, 배변, 목욕, 미용, 놀이, 훈련
- 건강 관리: 예방접종, 심장사상충/구충/외부기생충 예방, 약 복용, 체중, 증상, 병원 방문, 검사 기록
- 일정 관리: 반복 일정, 예약 일정, 추적 관찰 일정, overdue 처리
- 기록 관리: 타임라인, 문서 보관, 사진, 메모, 공유
- 협업 관리: 가족, 펫시터, 병원, 미용실, 호텔/데이케어와의 정보 전달
- 확장 가능성: 웨어러블, 위치 추적, 보험/결제, 커머스, 커뮤니티, AI 보조 기능

## 2. 핵심 결론

반려동물 건강/생활 관리 앱의 본질은 “한 번 입력한 반려동물 정보를 중심으로, 반복적으로 발생하는 관리 행위를 잊지 않도록 돕고, 누적 기록을 나중에 다시 꺼내 쓸 수 있게 만드는 것”이다.

시장에 보이는 공통 패턴은 다음과 같다.

- 대부분의 앱은 `반려동물 프로필 + 기록 + 리마인더 + 병원/돌봄 정보 공유` 조합을 기본 축으로 삼는다.
- 사용자가 진짜로 불편해하는 지점은 “정보가 종이, 병원 문자, 가족 카톡, 캘린더, 머릿속 기억으로 흩어져 있다”는 점이다.
- 건강 관리 기능은 강력한 진단 도구보다 `조기 이상 징후 감지`, `주기 관리`, `기록 보관`, `수의사에게 보여주기 쉬운 형태`가 실제 가치가 높다.
- 생활 관리 기능은 기록 피로도가 높기 때문에, “모든 것을 다 기록”하는 방식보다 `반복 루틴 자동화 + 빠른 체크 + 예외 상황 기록`이 더 현실적이다.
- 알림은 많을수록 좋은 것이 아니라, `놓치면 위험한 것`과 `습관 형성용`을 구분해야 유지율이 올라간다.
- 다펫 가구, 가족 공동 양육, 여행/출장 시 위탁 돌봄, 만성질환/복약 관리 반려동물은 일반 보호자보다 훨씬 높은 니즈를 가진다.

## 3. 주요 사용자 유형

### 3.1 1차 사용자: 일반 반려인

가장 넓은 사용자군이다.

- 개/고양이 1마리를 키우는 보호자
- 예방접종, 급여, 산책, 목욕, 미용 등 기본 관리가 핵심
- “기억 의존 관리”에서 “앱 기반 관리”로 옮기려는 니즈가 있음
- 진입 장벽은 낮아야 하고 입력 스트레스가 작아야 함

주요 목표:

- 해야 할 일을 잊지 않기
- 최근 기록을 빨리 확인하기
- 병원 갈 때 필요한 정보 찾기

### 3.2 초보 보호자

입양 직후 또는 어린 강아지/고양이를 처음 키우는 사용자다.

- 무엇을 언제 해야 하는지 모름
- 백신, 중성화, 급여량, 사회화, 훈련, 목욕/미용 주기에 대한 불안이 큼
- “관리 도구”보다 “가이드 포함 관리 도구”에 더 큰 가치를 느낌

주요 목표:

- 표준 체크리스트를 따르고 싶음
- 놓치면 안 되는 일정만 정확히 알고 싶음
- 반려동물 연령/종류별 기본 루틴을 받고 싶음

### 3.3 다펫 가구

2마리 이상을 함께 관리하는 사용자다.

- 개별 개체마다 일정, 약, 알러지, 병력, 사료, 병원이 다를 수 있음
- 실수가 발생하면 비용과 위험이 큼
- 앱 전환 비용이 높지만 정착하면 충성도가 높음

주요 목표:

- 펫별 스위칭이 빠를 것
- 같은 종류 일정도 개별 펫 상태에 맞게 관리할 것
- 한 화면에서 “누가 무엇이 due인지” 보여줄 것

### 3.4 만성질환/복약 관리 보호자

앱의 고가치 사용자 후보군이다.

- 신장 질환, 당뇨, 피부 질환, 알러지, 심장 질환, 재활 관리 등
- 단순 리마인더가 아니라 “실행 여부 기록”과 “증상 변화 추적”이 중요
- 병원과의 정보 공유성이 중요함

주요 목표:

- 약을 언제 얼마나 줬는지 남기기
- 증상, 식욕, 음수량, 체중 변화를 추적하기
- 병원에 보여줄 요약 자료 만들기

### 3.5 가족 공동 양육 사용자

한 사람이 아닌 여러 사람이 함께 돌보는 유형이다.

- 가족, 배우자, 룸메이트, 도우미, 산책 대행인 등이 관여
- “누가 이미 했다”가 공유되지 않으면 중복 급여/중복 투약 위험이 있음

주요 목표:

- 실시간 상태 공유
- 담당자 할당
- 실행 완료 체크와 히스토리 확인

### 3.6 외부 돌봄 연계 사용자

여행, 출장, 장시간 외출 시 펫시터/호텔/위탁돌봄을 쓰는 사용자다.

- 낯선 사람에게 정확한 돌봄 정보를 전달해야 함
- 약, 식이, 행동 특성, 응급 연락처가 중요

주요 목표:

- 돌봄 요약 프로필 공유
- 금기 사항과 주의 행동 전달
- 최근 건강 상태와 투약 이력 전달

### 3.7 고관여 건강 추적 사용자

웨어러블, GPS, 활동량, 수면, 이상 행동 추적에 관심이 높은 사용자다.

- 데이터 기반 관리 선호
- 반려동물의 이상 패턴을 조기에 알고 싶음
- 건강 데이터와 일상 데이터를 연결해 보고 싶어 함

주요 목표:

- 활동/수면 변화 감지
- 평소 패턴 대비 이상 알림
- 수의사 상담 전 근거 자료 확보

## 4. 핵심 문제 시나리오

### 4.1 “언제 맞았는지 기억이 안 난다”

대표 사례:

- 예방접종 날짜를 종이 수첩이나 병원 문자에서 찾아야 함
- 심장사상충/구충약을 지난달에 먹였는지 헷갈림
- 벼룩/진드기 예방 주기가 섞여 있음

문제:

- 보호자가 기억에 의존
- 기록 위치가 분산
- due/overdue 상태가 직관적으로 보이지 않음

필요한 해결:

- 마지막 실행일과 다음 예정일 자동 계산
- due/overdue 뱃지
- 주기 변경 가능

### 4.2 “병원에 갔는데 자료가 없다”

대표 사례:

- 백신 수첩을 안 가져옴
- 최근 체중, 증상 시작일, 약 이름이 기억나지 않음
- 이전 검사 결과 PDF/사진을 찾지 못함

문제:

- 병원에서 설명 누락
- 중복 검사, 잘못된 문진 가능성 증가

필요한 해결:

- 병원 방문용 요약 카드
- 문서/사진 첨부
- 최근 증상 및 약 복용 이력 타임라인

### 4.3 “가족이 이미 밥 줬는지 모르겠다”

대표 사례:

- 아침 급여를 누가 했는지 모름
- 약을 두 번 주거나 아예 빼먹음
- 산책/배변 처리 여부가 공유되지 않음

문제:

- 공동 양육에서 기록이 동기화되지 않음
- 실행 완료 기록과 담당자 정보 부재

필요한 해결:

- 체크형 루틴
- 실행자 표시
- 최근 완료 내역 실시간 공유

### 4.4 “여행 가는데 펫시터에게 설명할 게 너무 많다”

대표 사례:

- 사료 양, 급여 시간, 약 복용법, 분리불안, 산책 습관, 병원 연락처를 따로 정리해야 함
- 구두 설명 누락 위험

문제:

- 위탁 돌봄 시 정보 전달 비용이 큼
- 건강/행동 정보가 표준화되지 않음

필요한 해결:

- 공유 가능한 펫 프로필
- 주의 행동/복약/응급 정보 요약
- 임시 접근 링크 또는 PDF 공유

### 4.5 “무언가 이상한데 정확히 어떻게 다른지 설명이 안 된다”

대표 사례:

- 평소보다 덜 먹음
- 물을 많이 마시는 것 같음
- 잠을 더 많이 자거나 덜 움직이는 것 같음
- 긁기, 핥기, 짖기 증가

문제:

- 보호자의 체감은 있으나 기록이 없어 추세 설명이 어려움
- 병원 진료 시 근거가 약함

필요한 해결:

- 증상/행동/섭취량/체중 로그
- 전주/평소 대비 변화 표시
- 메모와 사진 첨부

### 4.6 “기록하려다 귀찮아서 앱을 안 쓰게 된다”

대표 사례:

- 입력할 항목이 너무 많음
- 사소한 일까지 전부 기록하려니 유지가 안 됨
- 알림이 너무 많이 와서 꺼버림

문제:

- 사용 유지율을 해치는 가장 큰 원인

필요한 해결:

- 빠른 입력
- 기본 루틴 자동 생성
- 중요한 것 중심의 알림 설계

## 5. 필수 기능

### 5.1 반려동물 프로필

최소 필요 정보:

- 이름
- 종
- 품종 또는 혼합종
- 성별
- 중성화 여부
- 생년월일 또는 추정 나이
- 몸무게/목표 몸무게
- 사진
- 마이크로칩 번호
- 등록번호
- 주치의/병원 정보
- 특이사항: 알러지, 복용 중 약, 주의 행동, 식이 제한

이 기능이 중요한 이유:

- 거의 모든 기록과 일정의 기준 엔티티가 됨
- 공유와 요약 기능의 출발점이 됨

### 5.2 일정/리마인더 관리

반드시 필요한 일정 유형:

- 예방접종
- 구충/심장사상충/외부기생충 예방
- 약 복용
- 병원 예약
- 미용/목욕/발톱/양치 등 관리 일정
- 급여/산책 같은 반복 루틴

필수 요건:

- 단건 일정
- 반복 일정
- custom 주기
- 완료 처리
- 미루기
- overdue 표시
- 다음 일정 자동 생성

### 5.3 건강 기록

핵심 기록 항목:

- 접종 기록
- 투약 기록
- 병원 방문 기록
- 증상 기록
- 체중 기록
- 검사/처방 문서 보관
- 수술/시술 이력

필수 요건:

- 날짜 기반 타임라인
- 첨부파일/사진
- 메모
- “최근 기록” 빠른 조회

### 5.4 생활 기록

핵심 기록 항목:

- 급여
- 물 섭취 관련 메모
- 산책
- 배변
- 목욕/미용
- 훈련/행동 메모

필수 요건:

- 초간단 체크인 방식
- 반복 루틴과 연결
- 예외 상황 메모

### 5.5 대시보드

사용자가 앱을 열었을 때 바로 확인해야 하는 것:

- 오늘 해야 할 일
- 곧 다가오는 일정
- overdue 항목
- 최근 기록
- 주의가 필요한 반려동물

### 5.6 기록 공유/내보내기

최소한 다음은 필요성이 높다.

- 병원 방문용 요약
- 예방접종/투약/병력 내역 export
- 이미지/PDF 첨부 보관

## 6. 있으면 좋은 기능

### 6.1 스마트 추천

- 반려동물 연령/종류에 따른 기본 관리 루틴 제안
- 강아지/고양이 성장 단계별 체크리스트
- 기록 누락 탐지

주의:

- 의학적 판단처럼 보이지 않도록 설계 필요

### 6.2 가족/공동 양육 협업

- 멤버 초대
- 역할 분리
- 누가 완료했는지 히스토리
- 중복 수행 방지

### 6.3 펫시터/위탁돌봄용 공유 모드

- 제한된 읽기 전용 공유
- 임시 링크
- 돌봄 매뉴얼
- 응급 연락처/병원 정보 강조

### 6.4 문서 인식/자동 구조화

- 접종 수첩 사진 업로드
- 영수증/검사 결과 문서 보관
- OCR 기반 항목 추출

주의:

- 자동 인식 오류 검수 플로우 필요

### 6.5 웨어러블/위치 추적 연동

- 활동량
- 수면
- GPS
- 이상 행동
- 휴식 시 심박/호흡 추세

### 6.6 비용 추적

- 병원비
- 사료/간식
- 미용/호텔/시터
- 월간/카테고리별 지출

### 6.7 성장/건강 리포트

- 월간 요약
- 체중 추세
- 예방 관리 이행률
- 병원 방문 빈도

### 6.8 다종 반려동물 대응

- 개/고양이 외 소동물/조류/파충류
- 종별 다른 주기/지표 관리

## 8. 알림/일정/기록 기능의 일반적인 패턴

### 8.1 알림 패턴

반려동물 앱에서 알림은 보통 세 부류로 나뉜다.

#### 1) 예정 알림

아직 due 전이지만 미리 알려주는 알림이다.

예시:

- 예방접종 30일 전, 7일 전
- 병원 예약 7일 전, 2일 전
- 미용 예약 하루 전

장점:

- 준비 시간을 줌

주의:

- 너무 이르면 무시되고, 너무 늦으면 실효성이 떨어짐

#### 2) 실행 알림

정해진 시점에 바로 행동을 요구하는 알림이다.

예시:

- 오전 8시 약 복용
- 오후 7시 급여
- 밤 10시 양치

장점:

- 생활 루틴 형성에 좋음

주의:

- 너무 잦으면 피로도가 매우 높음

#### 3) overdue 알림

이미 지나간 일정에 대한 추적 알림이다.

예시:

- 심장사상충 예방 overdue 3일
- 백신 overdue 1주

장점:

- 놓친 일정 회수에 효과적

주의:

- 죄책감 유발형 UX가 되면 이탈 가능성 증가

### 8.2 일반적인 알림 설계 원칙

- 여러 알림 채널이 있어도 앱 내부 기준 상태는 하나여야 한다.
- `due`, `upcoming`, `completed`, `skipped`, `overdue` 상태가 명확해야 한다.
- 일정 성격에 따라 알림 강도를 달리해야 한다.
- 주기성 예방 관리와 데일리 루틴은 다른 UX가 필요하다.
- 사용자가 알림을 받는 것보다 “앱 안에서 왜 이 알림이 왔는지 이해”하는 것이 더 중요하다.

### 8.3 일정 모델링 패턴

반려동물 관리 일정은 보통 두 가지 모델이 섞인다.

#### 계획 기반

“앞으로 할 일” 중심이다.

예시:

- 다음 심장사상충약: 4월 20일
- 다음 미용: 5월 2일

장점:

- 캘린더/할 일 UX에 적합

단점:

- 실제 실행 기록이 분리되지 않으면 신뢰도가 낮아짐

#### 실행 기반

“실제로 했다” 중심이다.

예시:

- 3월 20일 심장사상충약 투여 완료
- 3월 22일 산책 28분

장점:

- 사실 기록으로서 강함

단점:

- 다음 일정 생성 규칙이 없으면 리마인더가 약함

실제 서비스는 대개 다음 구조가 좋다.

- 계획 엔티티: 앞으로 해야 할 것
- 실행 로그: 실제로 한 것
- 완료 시 다음 계획 자동 생성

### 8.4 기록 UI 패턴

가장 자주 보이는 패턴은 아래와 같다.

#### 타임라인 중심

- 병원, 증상, 약, 체중, 문서를 시간순으로 확인
- “최근 무슨 일이 있었는지” 회상에 강함

#### 카테고리 탭 중심

- 접종 / 투약 / 체중 / 산책 / 배변 / 미용 등으로 분리
- 특정 기록만 빠르게 찾기 좋음

#### 오늘 중심

- 오늘 해야 할 일
- 오늘 완료한 일
- 오늘 특이사항

가장 실용적인 조합은 다음이다.

- 홈은 오늘 중심
- 상세는 타임라인 중심
- 조회는 카테고리 필터 제공

### 8.5 빠른 기록 패턴

유지율을 위해 중요하다.

- 한 탭 완료
- 최근 사용한 기록 재사용
- 기본값 자동 입력
- 템플릿 기반 빠른 추가
- 사진 후 메모 나중 입력

### 8.6 공유 패턴

실무적으로 자주 필요한 공유는 세 가지다.

- 병원용 요약: 최근 증상, 약, 접종, 문서
- 시터용 요약: 급여, 산책, 행동 주의, 응급 연락처
- 여행/이동용 문서: 접종 기록, 식별 정보, 병원 연락처

공유 방식 후보:

- 앱 내 멤버 초대
- 읽기 전용 링크
- PDF export
- 이미지 카드 export

## 9. MVP 범위 제안

이 섹션은 “무엇이 가장 작은 제품 단위로 의미 있는가”를 연구 관점에서 제안한 것이다. 최종 범위 결정은 사람이 해야 한다.

### 9.1 MVP가 해결해야 할 핵심 문제

최소한 아래 두 가지는 반드시 해결해야 한다.

- 반려동물 관리 일정과 기록이 흩어져 있는 문제
- 중요한 일정과 복약을 잊는 문제

### 9.2 MVP에 포함할 만한 범위

#### A. 반려동물 프로필

- 기본 프로필
- 건강 특이사항
- 병원/응급 연락처

#### B. 일정/리마인더

- 예방 관리 일정 생성
- 약 복용 일정 생성
- 병원/미용 등 단건 일정 생성
- 반복 규칙
- 완료/미루기/overdue

#### C. 기록

- 접종
- 투약
- 병원 방문
- 체중
- 자유 메모

#### D. 홈 대시보드

- 오늘 할 일
- 곧 다가오는 일정
- 최근 기록

#### E. 기본 공유

- 병원 방문용 요약 export 또는 읽기 화면

### 9.3 MVP에서 제외를 고려할 기능

초기 복잡도를 크게 높이는 항목이다.

- 웨어러블/IoT 연동
- GPS 실시간 추적
- AI 진단형 기능
- 커뮤니티
- 커머스
- 보험 청구 자동화
- 결제/예약 마켓플레이스
- 고도화된 멀티유저 권한 체계

### 9.4 MVP 성공 판단 기준 후보

- 보호자가 1주 이상 반복 일정 1개 이상 유지하는가
- 완료 체크와 기록 남기기가 귀찮지 않은가
- 병원 방문 전 필요한 정보를 1분 안에 찾을 수 있는가
- overdue 상태가 명확히 보이는가

## 11. 제품 설계 시 주의해야 할 함정

### 11.1 기록 과부하

모든 것을 세밀하게 기록하게 하면 초기 인상은 좋아 보여도 장기 유지율이 낮아질 가능성이 크다.

권장 관점:

- 꼭 필요한 기록을 먼저 정의
- 나머지는 선택적 확장으로 남김

### 11.2 의료 서비스처럼 보이는 표현

건강 앱은 사용자 기대를 쉽게 의료 판단 쪽으로 끌어올린다.

주의:

- “이상 가능성 알림”과 “진단”을 분리
- 수의사 상담 유도 문구 필요

### 11.3 종별 차이 무시

개와 고양이도 관리 패턴이 다르고, 소동물/조류/파충류는 더 크게 다르다.

주의:

- 초기 타겟 종을 명확히 해야 함
- 범용 데이터 모델과 종별 템플릿을 분리하는 것이 유리

### 11.4 알림 피로

루틴 알림을 많이 켜면 사용자가 전체 알림을 꺼버릴 수 있다.

권장 관점:

- 놓치면 위험한 일정 우선
- 생활 루틴은 체크리스트나 위젯형 UX도 검토 가치가 있음

### 11.5 공유 데이터의 최신성 문제

시터/가족/병원에 공유한 정보가 최신인지 보장되지 않으면 오히려 위험할 수 있다.

주의:

- 마지막 업데이트 시각
- 공유 대상별 표시 범위
- 만료 링크 또는 스냅샷 구분 필요

## 12. 사람이 이후 결정해야 할 주요 선택지

이 문서는 결정이 아니라 선택지를 정리한다.

### 12.1 1차 타겟

후보:

- 일반 반려인 전체
- 초보 보호자 중심
- 건강/복약 관리 중심
- 다펫/가족 공동 양육 중심

### 12.2 1차 종 범위

후보:

- 개 중심
- 개/고양이 공통
- 다종 확장 전제

### 12.3 제품 포지션

후보:

- 리마인더 중심 생활 관리 앱
- 의료 기록 중심 건강 관리 앱
- 공동 양육 운영 앱
- 웨어러블 연계 건강 모니터링 앱

### 12.4 기록 깊이

후보:

- 최소 기록형: 체크 위주
- 중간 기록형: 메모+첨부
- 고정밀 기록형: structured health log 중심

## 14. 최종 요약

반려동물 건강/생활 관리 앱의 핵심은 기능 수를 늘리는 것이 아니라 다음 세 가지를 한 구조 안에 안정적으로 묶는 것이다.

- 반려동물별 단일 정보 원본
- 반복 일정과 실제 실행 기록의 연결
- 병원/가족/돌봄 상황에서 다시 꺼내 쓰기 쉬운 공유 가능한 기록

가장 현실적인 초기 제품 가치는 `프로필 + 일정/리마인더 + 핵심 건강 기록 + 빠른 생활 체크 + 요약 공유`에서 나온다. 이후 협업, 문서 자동화, 웨어러블, 비용 추적, 건강 인사이트로 확장하는 구조가 자연스럽다.
